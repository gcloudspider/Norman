####职责链模式:
    使多个对象都有机会处理请求,从而避免请求的发送者和接收者之间耦合关系,将这些对象连成一条链,并沿着这条链传递该请求,直到有一个对象处理它为止

    请求的处理形成一条链,直到有一个对象处理请求!


####职责链使用场合:
    1.有多个的对象可以处理一个请求,由哪个对象处理该请求是运行时刻自动确定的
    2.如果想在不明确指定接收者的情况下,向多个对象中的一个提交一个请求
    3.可以处理一个请求的对象集合应被动态指定
